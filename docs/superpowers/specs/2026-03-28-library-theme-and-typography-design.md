# 文库首页主题与字体统一设计

**日期**: 2026-03-28
**状态**: 已确认
**范围**: 文库首页/列表页主题系统、全站主字体统一、首页图标体系收敛

---

## 一、背景与目标

当前文库首页存在三个直接影响产品质感的问题：

1. 文档卡片之间缺少足够的节奏与留白，列表更像原型态页面，而不是已经打磨过的产品。
2. 文章正文使用的 serif 字体比应用 UI 现有字体更有质感，但两套字体并存，导致产品语言不统一。
3. 首页底部导航仍然使用 emoji，而智读页已经切到了 Lucide 图标体系，因此用户会感觉同一产品内存在两套图标语言。这个问题不是缓存问题，而是源码现状：`app/(reader)/layout.tsx` 仍写死了 emoji，`app/(reader)/[id]/smart/page.tsx` 已使用 Lucide 图标。

本次改动的目标：

- 为文库首页/列表页建立可扩展的主题系统。
- 首批交付两套体验：`留白型` 与 `杂志型`。
- 默认主题为 `留白型`。
- 主题切换使用抽屉交互，并支持后续继续增加新体验。
- 全站主文字统一到文章当前使用的 serif 字体气质。
- 首页导航图标与智读页统一到同一套 Lucide 图标体系。

---

## 二、范围与非目标

### 2.1 本次范围

- 文库首页/列表页的搜索区、标签区、文档卡片、分页节奏。
- 首页专属主题抽屉与主题持久化。
- Reader 应用中的公共字体统一。
- 首页底部导航与相关入口图标统一。

### 2.2 非目标

- 不修改文章页与智读页的布局结构。
- 不让首页主题切换影响文章页与智读页的视觉主题。
- 不重做全局 dark mode 逻辑。
- 不在本轮引入第三套及以上主题，只预留扩展能力。

---

## 三、用户决策记录

用户已确认以下产品决策：

- 全站主文字统一采用文章当前字体体系。
- 首页主题切换仅作用于文库首页/列表页，不作用于文章页和智读页。
- 首批同时提供两套体验：`留白型` 与 `杂志型`。
- 主题切换交互采用右侧抽屉。
- 默认主题选择 `留白型`。

---

## 四、方案比较与选择

### 方案 A：轻量参数切换

仅调整卡片间距、圆角、阴影、标题字号等少数参数。

优点：

- 实现最简单。
- 回归风险最低。

缺点：

- 主题感弱。
- 更像“同一页面的两组参数”，而不是“两个可感知的浏览体验”。

### 方案 B：结构化主题切换

为首页建立独立的 `library theme tokens`，由主题统一控制搜索区、标签区、卡片密度、标题层级、摘要存在感、分页节奏。

优点：

- 主题差异明确。
- 扩展第三套主题时成本低。
- 可严格限制在首页作用域内，不影响文章页和智读页。

缺点：

- 需要建立明确的主题边界和局部状态。

### 方案 C：深度布局级主题

除了视觉 token，还让不同主题拥有不同的首页布局骨架，例如搜索区位置和列表骨架也不同。

优点：

- 表现力最强。

缺点：

- 复杂度高。
- 回归风险高。
- 当前需求阶段没有必要。

**最终选择：方案 B。**

## 五、主题系统设计

### 5.1 主题集合

首批提供两个首页主题：

- `airy` / 留白型
- `magazine` / 杂志型

### 5.2 主题定位

#### 留白型

目标是做出更安静、更克制、更像高质量阅读产品的首页体验。

视觉特征：

- 更大的卡片间距与垂直节奏。
- 更柔和的边框与更轻的表面层级。
- 标题存在感高，但不过度侵略。
- 摘要更克制，强调呼吸感与浏览舒适度。
- 标签更轻，辅助而不抢内容。

这是默认主题。

#### 杂志型

目标是强化“内容策展感”和“编辑感”。

视觉特征：

- 标题更强、更有封面感。
- 摘要更明确，适合内容导读。
- 标签和信息层级更鲜明。
- 列表节奏更紧凑，整体更像刊物编排。

### 5.3 主题作用范围

主题只作用在文库首页/列表页的以下模块：

- 顶部搜索区
- 标签筛选区
- 文档卡片
- 分页与列表节奏

主题不作用于：

- 文章详情页
- 智读页
- Reader 公共结构以外的页面骨架

## 六、交互设计

### 6.1 主题入口

在文库首页顶部区域提供一个“主题”入口，点击后从右侧拉出抽屉。

该入口应只在文库首页/列表页可见；文章页与智读页不展示该入口。

### 6.2 主题抽屉

抽屉内容包括：

- 当前主题说明
- `留白型` 主题卡片
- `杂志型` 主题卡片
- 未来主题的扩展占位方式

交互要求：

- 点击主题卡片后立即切换首页视觉。
- 当前选中项有明确状态。
- 抽屉可关闭，不阻塞首页内容。
- 主题切换后立即可见，不需要刷新。

### 6.3 持久化

主题选择保存在浏览器本地存储中。

规则：

- 首次访问且无本地记录时，默认使用 `airy`。
- 用户切换后写入 `localStorage`。
- 刷新页面或再次回到首页时恢复上次选择。

## 七、字体系统设计

### 7.1 总体策略

应用主文字统一采用文章当前的 serif 字体体系，以建立一致的产品语言。

建议主字体栈沿用文章阅读区当前策略：

`Georgia, 'Noto Serif SC', 'Source Han Serif SC', serif`

### 7.2 应用规则

统一到 serif 的对象：

- 顶部品牌文字
- 首页标题与摘要
- 文章页标题与正文
- 智读页的主要内容文本
- 常规说明文案

保留功能性辅助字体或样式的对象：

- 代码块与代码内联内容
- 需要 tabular 数字的版本号、计数、时间数据
- 极小的功能型标记（必要时允许沿用当前辅助字体类别）

### 7.3 需要修正的现状

当前文章页容器使用 `reading-body`，但文章标题又显式写了 `font-sans`，破坏了统一目标。本轮需要一并修正。

---

## 八、图标系统设计

### 8.1 问题定义

首页底部导航当前使用 emoji，智读页使用 Lucide 图标，导致图标语言不一致。

### 8.2 目标

首页、智读页以及相关导航入口统一使用 Lucide 图标体系。

### 8.3 处理策略

- 将首页底部导航中的 emoji 替换为 Lucide 图标。
- 图标语义与智读页保持一致。
- 图标大小、描边风格、对齐方式统一。

---

## 九、组件与结构调整

### 9.1 首页主题状态

首页主题状态应被限制在首页层，不进入全局 dark mode 或全局应用主题。

建议结构：

- 新增首页专属的 client-side 主题壳层或 hook。
- 由该层负责：
  - 读取与写入 `localStorage`
  - 维护 `data-library-theme`
  - 控制主题抽屉打开/关闭

### 9.2 首页页面结构

`app/(reader)/page.tsx` 负责：

- 挂载首页主题容器
- 将主题值传递给搜索区、标签区、卡片区和分页区
- 控制首页内容节奏

### 9.3 文档卡片

`components/DocumentCard.tsx` 需要支持主题分支，用于控制：

- 卡片间距
- 卡片圆角
- 边框与阴影
- 标题字号与层级
- 摘要展示密度
- 元信息与标签的视觉存在感

### 9.4 Reader 布局

`app/(reader)/layout.tsx` 需要处理两件事：

- 修复首页底部导航图标，替换 emoji
- 放置首页可见的主题入口，但不要让入口出现在文章页和智读页

## 十、实现建议

### 10.1 推荐数据形式

为首页定义轻量主题枚举：

```ts
type LibraryTheme = 'airy' | 'magazine'
```

并建立对应的主题 token 映射，用于集中维护样式差异。

### 10.2 推荐样式组织

优先使用以下组合：

- 页面根容器挂 `data-library-theme`
- Tailwind class 分支控制主要差异
- 必要时在 `globals.css` 中添加少量首页专属 token 或 utility

不建议把首页主题直接塞进当前全局颜色 token，因为会扩大影响范围。

### 10.3 推荐抽屉形态

优先复用现有 Radix/Dialog 基础设施实现侧边抽屉，而不是再引入新交互系统。

原因：

- 项目中已存在 dialog 组件
- 可减少额外依赖和回归面
- 更利于统一关闭行为、焦点管理和移动端表现

## 十一、验证策略

### 11.1 功能验证

- 首页打开时默认主题为 `留白型`
- 抽屉可正常打开、关闭、切换主题
- 切换后刷新页面仍保留选择

### 11.2 视觉验证

- 首页卡片之间存在明确间距
- `留白型` 与 `杂志型` 的差异肉眼可见
- 搜索区、标签区、卡片和分页在两套主题下都成立

### 11.3 范围隔离验证

- 文章页不受首页主题切换影响
- 智读页不受首页主题切换影响
- dark mode 逻辑保持原样

### 11.4 一致性验证

- 首页和智读页图标语言一致
- 文章页、首页、智读页主文字统一到同一 serif 气质
- 移动端底部导航不再出现 emoji 图标

### 11.5 设备验证

- 桌面端首页
- 移动端首页
- 文章页
- 智读页

## 十二、风险与缓解

### 风险 1：字体统一后控件可读性下降

缓解：

- 仅统一主文字气质
- 对极小功能字、代码、tabular 数字保留辅助样式

### 风险 2：主题逻辑扩散到非首页页面

缓解：

- 首页主题状态局部化
- 抽屉入口仅在首页/列表页可见
- 不修改文章页与智读页布局结构

### 风险 3：首页主题差异不足

缓解：

- 主题差异不仅体现在颜色，也体现在节奏、层级、摘要密度和标签表现

---

## 十三、实施顺序

1. 修正字体基础层，明确 serif 主字体与例外项。
2. 统一首页底部导航与相关入口图标。
3. 建立首页主题状态与抽屉交互。
4. 改造首页搜索区、标签区、文档卡片、分页区以支持主题分支。
5. 做浏览器验证，确保主题隔离、字体统一和图标一致。
